Architecture

How Finch thinks

A bounded multi-agent loop routes every question to the right specialist.

1

You ask a question

Any financial question — market data, your portfolio, concepts, or a blend of all three.

↓
2

Orchestrator classifies intent

The Orchestrator Agent reads your message and decides which specialist agents to invoke — via tool-calling.

↓
3

Specialist agents act

Stock Agent fetches live market data. Portfolio Agent queries ChromaDB for your holdings. Both can run sequentially.

↓
4

LLM synthesizes the answer

Results return to the LLM, which generates a coherent, grounded natural-language response.

⚡

A complex query like "Should I buy more NVDA given my current position?" triggers up to 6 LLM calls across the Orchestrator, Portfolio Agent, and Stock Agent — all transparently, in under seconds.
